# Career Doomsday Clock — UX 개선 계획

## 1. 입력 개선 (설문 폼)

### 1.1 연령대 필드 추가 ✅ 즉시 적용
- 연령대 선택 드롭다운 추가 (10대/20대/30대/40대/50대 이상)
- Bedrock Agent 프롬프트에 연령대 반영하여 분석 정확도 향상

### 1.2 취미 입력 가이드 ✅ 즉시 적용
- placeholder에 "예: 독서, 요리, 코딩" 형태로 쉼표 구분 안내
- 쉼표(,) 구분을 기본으로 하되, 자유 텍스트도 허용 (Agent가 자연어 처리)

### 1.3 직업 입력 가이드 ✅ 즉시 적용
- placeholder에 "예: 프론트엔드 개발자, 학생, N잡러" 등 다양한 예시 제공
- "학생", "무직", "N잡러" 등 비전형 직업도 자연스럽게 입력 가능하도록 안내

### 1.4 희망 은퇴 시기 필드 추가 ✅ 즉시 적용
- "언제까지 일하고 싶으신가요?" 선택형 (5년 이내 / 10년 이내 / 20년 이내 / 평생 현역)
- D-Day 계산 시 참고 데이터로 활용

## 2. 개인정보 처리 ✅ 즉시 적용

### 2.1 개인정보 안내 문구
- 설문 폼 하단에 개인정보 처리 안내 추가
- "입력된 정보는 분석 목적으로만 사용되며, 세션 종료 시 자동 삭제됩니다"
- 이름은 분석에만 사용되고 결과에 표시되지 않음을 명시

## 3. 세션 복귀 (재방문 처리)

### 3.1 현재 구조
- sessionStorage 기반이므로 탭/브라우저 닫으면 세션 소멸
- 이는 개인정보 보호 측면에서 의도된 설계

### 3.2 향후 고려사항 (현재 미적용)
- localStorage로 전환 시 재방문 복귀 가능하나 개인정보 이슈
- URL에 세션 ID 포함하여 공유 가능하게 하는 방안 검토 가능
- 현재는 ephemeral session 유지가 보안 가이드라인에 부합

## 4. 프로세스 편의성 ✅ 즉시 적용

### 4.1 에러 시 재시도
- 분석 실패 시 로딩 화면에서 "다시 시도" 버튼 제공
- 설문 데이터를 sessionStorage에 보관하여 재입력 불필요

### 4.2 결과 페이지에서 재시도
- 결과 페이지에 "다시 분석하기" 버튼 추가

## 5. 아웃풋 개선 ✅ 즉시 적용

### 5.1 스킬 위험도 정렬 및 강조
- replacement_prob 기준 내림차순 정렬 (가장 위험한 스킬 먼저)
- 최고 위험 스킬에 "CRITICAL" 뱃지 표시

### 5.2 커리어 카드 추천 강조
- 첫 번째 카드를 "TOP PICK" 으로 강조 표시
- "이런 직업이 있구나" 보다 "이 직업을 추천합니다" 느낌으로

### 5.3 로드맵 구체화
- 각 단계에 학습 리소스 링크 안내 텍스트 추가
- "관련 교육 과정을 검색해보세요" 등의 액션 유도

### 5.4 결과 공유/저장
- 결과 페이지에 "결과 캡처하기" 안내 (브라우저 기본 기능 활용)

## 6. 향후 수익 구조 고려 (참고용)

### 6.1 교육 플랫폼 연계
- 로드맵 단계별 관련 교육 사이트 링크 (Coursera, Udemy 등)
- 향후 제휴 마케팅/광고 수익 가능성

### 6.2 기업 B2B
- 기업 HR 부서 대상 대량 분석 서비스
- 팀 단위 스킬 갭 분석 리포트
